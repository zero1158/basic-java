自定义泛型
泛型类:
    自定义泛型的子类若是在继承时不指定,则泛型也被继承
    构造器的时候不加<>,实例化的时候要加
    泛型不同的引用不能相互赋值
    静态方法中不能使用泛型:
            泛型需要在实例化(创建对象)的时候指定,但是静态方法的加载要早于实例化:--->重要
    异常类不能声明为泛型的

泛型方法:
    在方法中出现了泛型结构,泛型参数于类的泛型参数没有关系-->泛型方法所属的类是不是泛型的都没有关系
        public <E> List<E> getList(E[] arr)
        同名同参数的方法,在不同的类中,要么都是static的,要么都不是

    继承关系: 见图
        虽然类A是类B的父类,但是G<A>类和G<B>类是并列关系 : 如List<A> 和List<B>是并列的两个类,不能做相互赋值操作
                          补充: A<G> 是 B<G>的父类
    通配符(?)的使用 : 作为公共父类使用
        类A是类B的父类,但是G<A>类和G<B>类是并列关系,G<?> 作为其公共父类
